## 1. 企业主办方信息

### 1.1 机构画像
- **主办**：共青团上海市委员会，覆盖上海各区高校；联合主办：上海市人工智能行业协会（会员企业200+家）

### 1.5 合规/IP 规则

**开源协议声明**：参赛作品默认采用 **MIT License** 发布，参赛者保留署名权。参赛者可自行选择 Apache 2.0 或 GPL 3.0，但须在提交时明确声明。

**参赛者署名保留**：获奖作品在官方宣传材料中须注明参赛者姓名；如未来作品用于展示，须获参赛者书面同意方可修改。

**参赛资格**：全员须为中国大陆在读本科生或硕士研究生（含应届），须提供学信网在读证明或学校邮箱验证。每支队伍可设1名校外指导教师（不计入参赛人数）。

```
     通知其所在高校；平台更新规则，明确"指导教师 commit 贡献不得超过5%"。
```

## 5. 异常与边界案例库

### 案例 5.1：大众投票刷票（参见 R-03）
文化创意赛道大众投票期间，某队伍（"星云创作"）的投票在48小时内激增至2,340票，增速异于同期其他队伍（平均日增120票），来源IP集中于某高校宿舍IP段。平台风控降权处理，通知主办方。主办方问询后，该队承认组织班级同学集体投票（虽为真实账号，但超出平台鼓励的"自然传播"范畴）。处置：大众投票分数降权50%；公示期延长48小时并发布处置声明，不公开具体队伍名称，仅说明原则。

### 案例 5.2：指导教师代写（参见 4.7.2 申诉案例）
见上方申诉案例第2条。平台新增规则：提交前须填写"成员代码贡献比例声明"，并告知指导教师贡献≤5%的限制；未来将接入 GitHub API 自动统计贡献比例供评审参考。

### 案例 5.3：参赛资格造假（参见 R-08）
颁奖后，智慧校园赛道某三等奖队伍中有1名成员被举报已工作满1年（实为往届毕业生）。核查学信网记录：该成员已于2024年6月毕业，报名时填写"在读"。主办方取消该队获奖资格，奖项顺延；通知该成员所在原高校辅导员；奖金冻结未发放，直接转至下一名。

### 案例 5.4：内容违规（参见 R-06）
开发期，某参赛者在帖子中对其他城市队伍发表带有地域偏见的评论，被5名用户举报。平台内容安全模块1.5小时内标记，隐藏内容，向当事人发送警告+禁言72小时。发帖人删除内容并道歉后解禁。

### 案例 5.5：系统故障（参见 R-07）
06-13 提交截止日 22:00–22:45，图片/视频上传模块出现超时错误（成功率仅61%），影响约42支队伍。平台运维25分钟内修复，同时宣布提交截止时间延长至06-14 12:00（共计13小时延期窗口）。所有受影响队伍均完成重新提交，无因故障被排除。

## 7. 访谈提纲

### 7.1 发现与报名阶段

**主办方（团委/协会）侧**
1. 为什么选择竞赛模式而非培训工作坊？决策过程是什么样的？
2. 预算审批链路有多长？和企业主办相比，政府/团委的流程有什么特别之处？
3. 18所高校协办单位的管理和协调是怎么做的？有什么挑战？

**参赛者侧**
1. 您是从哪个渠道第一次知道这个比赛的？当时最吸引您的是什么？
2. 开源要求对您的参赛决定有影响吗？您觉得把代码开放出去有顾虑吗？
3. 在选择赛道时，您是怎么考虑的？是从自己擅长的出发，还是从感兴趣的问题出发？

### 7.2 参赛中（开发期）

**主办方侧**
1. 导师直播的效果如何？有没有评估指标？
2. 您最担心的风险是什么？在运营过程中有没有遇到让您始料未及的情况？

**参赛者侧**
1. 跨专业组队的体验怎么样？工程同学和设计/人文同学之间协作最难的是什么？
2. 您有没有在活动中认识了对你职业发展有影响的人（导师/队友/评审）？
3. 如果让您改进一件事，您会改进活动的哪个环节？

### 7.3 提交与评审

**主办方侧**
1. 大众投票的设计是您做的决定吗？现在回头看，您觉得20%/40%的权重合适吗？
2. 发现刷票行为后，您是怎么决定处置方式的？有没有内部争议？

**参赛者侧**
1. 您觉得评审标准足够清晰吗？在开始准备之前，您是否清楚评审会看哪些维度？
2. 如果您有申诉，评诉过程体验如何？

### 7.4 颁奖后与后续

**主办方侧**
1. 实习直推的12家企业，合作体验如何？企业觉得参赛者质量怎么样？
2. 这次活动对您的部门/机构有什么政治/形象层面的价值？

**参赛者侧**
1. 参赛后，您有没有把代码继续迭代？有没有其他人（非参赛）在用你们的开源成果？
2. 您会再次参加类似竞赛吗？什么因素会影响您的决定？
