## 4. 分阶段数据明细

### 4.1 创建活动

**活动标题**：AI驱动智能质检挑战赛 2025——视觉缺陷检测精度攻坚

**背景故事**：智海科技在汽车零部件客户的标书中，明确要求漏检率低于0.3%，但公司现有算法方案仍在1.2%徘徊。内部评估显示，达到该指标至少需要8个月研发周期和2名高级算法工程师的全时投入，而年度交付计划已无排期空间。开放挑战赛模式，能以更低成本、更短周期引入外部创新解法，同时探索多模态、合成数据增强等新技术路线的可行性。

**任务描述**：
- **技术赛道**：使用主办方提供的脱敏PCB产线图片数据集（8万张，含缺陷标注），训练视觉检测模型，使漏检率（miss rate）< 0.3%，误检率（false positive）< 2%，单张推理时间 < 50ms（RTX 3080 基准）。可使用任意开源框架，但不得直接调用付费商业API服务。
- **商业赛道**：基于技术方案，输出完整的商业部署方案，包含：总拥有成本（TCO）估算、ROI 测算（目标 >3x，回收期 < 18个月）、实施路径（分阶段方案）、风险点与缓解措施。

**FAQ（节选）**：
- Q：是否必须组队？A：不强制，1人也可参赛，但鼓励3–5人团队。
- Q：提供的数据集可以用于训练商业模型吗？A：不可以。数据集仅授权用于本次竞赛，提交后须删除本地副本，详见 NDA 第4条。
- Q：如果我的方案使用了开源模型权重，IP归属如何处理？A：开源模型权重按其原始协议使用，但你在竞赛中开发的改进代码、微调权重、方法论文档归智海科技所有。

## 5. 异常与边界案例库

### 案例 5.1：涉密数据误用
参赛者"独行侠·赵磊"在 GitHub public repo 中上传了含内部数据集路径和样本文件的 Jupyter Notebook，被平台内容扫描（关键词：`synnovator-internal`）在提交后4小时发现。平台立即冻结其账号，通知主办方安全负责人。主办方法务核实后发送书面警告，要求删除并签署补充说明函。最终确认无商业信息泄露，解冻账号，但记录 NDA 违约事件；若再次发生将取消资格。

### 案例 5.2：成果抄袭
终评前，评审林佳成发现某队（"轻骑兵"，最终未入围）的核心检测代码与 GitHub 某2024年开源项目结构高度一致，且无任何引用说明。平台指派独立核查，Copyleaks 报告显示代码相似度87%。通知该队伍48小时内自证原创。队伍承认大量复用未署名，主动退赛。处置：取消资格，发布处置公告（匿名，说明原则）。

### 案例 5.3：评审利益冲突
公示期第3天，有参赛者发现评审林佳成与"全栈检测"队长曾共同创业，在 LinkedIn 可见。主办方成立利益冲突核查小组，审查林佳成对全栈检测的打分（技术可行性91，高于其他评审均值5分）。决定：剔除林佳成对全栈检测的打分，按剩余3位评审重新加权（全栈检测加权总分从88.0降至87.2，排名不变）。发布声明，公开说明利益冲突核查结论。

### 案例 5.4：系统故障导致提交失败
提交截止日（05-28 23:00–23:30）期间，CDN 节点故障导致文件上传失败率升至41%。运维5分钟内告警，官方10分钟内发声明，提供24小时延期窗口（截止改为05-29 23:59）并开放邮件备用渠道。受影响的12支队伍全部完成补充提交，无一因故障被排除。

### 案例 5.5：知识产权侵权
公示期，第三方律所代理一家专利持有人，指出技术一等奖"量子跃迁"所使用的"合成数据增强对抗训练方法"与其持有的中国专利 ZL2023XXXXXXX 核心方案相同。主办方法务在5个工作日内完成专利检索。核实：专利申请日为2023年11月，量子跃迁的代码 commit 历史最早为2024年2月（竞赛启动前），且使用方式属于学术实验范畴，不构成商业侵权。向专利方出具书面说明，颁奖流程正常推进。

## 7. 访谈提纲

### 7.1 发现与报名阶段

**企业侧（主办方）**
1. 您是如何决定用悬赏竞赛替代内部研发的？有没有内部反对声音？
2. 预算审批过程是什么样的？决策链路有多长？
3. NDA 和 IP 归属条款是您自己起草的，还是法务主导？哪些条款有争议？
4. 在选择平台时，您最看重的3个因素是什么？

**参赛者侧**
1. 您从哪个渠道第一次看到这个活动？当时什么吸引了您点击进入？
2. 报名流程中，哪个步骤最令您犹豫（NDA？组队？数据集）？
3. 您有没有因为 NDA 或 IP 条款而考虑过放弃？

### 7.2 参赛中（开发期）

**企业侧**
1. 评审工作台满足您的需求吗？最大的痛点是什么？
2. 企业导师参与答疑的体验如何？有没有"回答了不该回答的内容"的担忧？

**参赛者侧**
1. 在开发过程中，您遇到的最大障碍是什么？是技术的还是信息的？
2. AMA 活动对您有帮助吗？您最希望活动组织方在开发期提供什么支持？
3. 您是怎么处理"不确定某个技术方案是否违反 NDA"的情况的？

### 7.3 提交与评审

**企业侧**
1. 您如何验证提交内容没有涉密数据？现有流程是否让您感到放心？
2. 评审维度是您自己设计的吗？现在回过头来看，有什么想改的？

**参赛者侧**
1. 您对评审结果有没有感到困惑或不满意的地方？如何让评审更透明？
2. 您觉得奖励（金额/类型）值得您投入的时间吗？

### 7.4 颁奖后与后续合作

**企业侧**
1. 这次活动最超出预期的地方是什么？最令您失望的呢？
2. 您会再次举办类似活动吗？会做哪些调整？
3. 与获奖团队的后续合作进展如何？

**参赛者侧**
1. 颁奖后您有没有收到主办方的后续联系？体验如何？
