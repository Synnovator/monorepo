# Output Template — Challenge Archive Structure

档案标题格式：
```
# 企业悬赏活动模拟档案：{活动名}（{scenario_name}）
```

以下为 0–7 章节的字段规范与内容要求。**严格按此顺序生成**。

---

## 第 0 章：一页摘要

一张表格，供访谈前快速引导受访者。

| 字段 | 内容 |
|------|------|
| 活动名称 | {challenge_theme 的正式名称} |
| 主办方 | {企业/机构名称} |
| 活动类型 | 企业悬赏 / 团委竞赛 |
| 核心目标 | 1–2句，说明主办方最想解决的问题 |
| 参与规模 | 访客 X / 报名 X 人 / 提交 X 队 / 获奖 X 队 |
| 总奖励金额 | ¥XXX，XXX / 等值资源 |
| 活动亮点 | 3–5 条 bullet，突出差异化卖点 |

---

## 第 1 章：企业主办方信息

包含以下小节：

**1.1 公司/机构画像**
- 企业（虚构）：行业 / 规模（人数）/ 成立年份 / 总部城市 / 业务方向摘要
- 团委：发文机构层级 / 覆盖地区 / 合作高校/协会列表

**1.2 活动负责人（虚构）**
- 姓名、职级、所属部门（如：开放创新事业部）

**1.3 发布动机**
- 2–3 条 bullet，说明为什么选择悬赏/竞赛模式而非内部研发

**1.4 预算结构**
| 项目 | 金额 | 说明 |
|------|------|------|
| 赛道A总奖池 | ¥XX | ... |
| 赛道B总奖池 | ¥XX | ... |
| 运营/评审费用 | ¥XX | 外部评委劳务等 |
| 总预算 | ¥XX | |

**1.5 合规/IP 规则**
- **Enterprise**：NDA 摘要（涉密范围 / 禁止外传 / 违约责任）；IP 归属条款（成果所有权声明）；数据合规说明
- **Youth-league**：开源协议声明（MIT / Apache 2.0 等）；参赛者保留署名权说明

## 第 3 章：全流程时间线

| 阶段 | 开始日期 | 结束日期 | 关键事件 | 平台动作 | 对外通知 |
|------|---------|---------|---------|---------|---------|
| 创建与审核 | T+0 | T+3 | 主办方提交活动草稿 | 系统自动审核+人工复核 | 无 |
| 报名期 | T+7 | T+21 | 官方发布公告；报名开放 | 推荐系统推送；邮件通知 | 公告+社媒 |
| 开发期 | T+22 | T+63 | AMA第1场（T+30）；AMA第2场（T+50）| 运营互动；官方答疑 | 进度提醒 |
| 提交期 | T+64 | T+70 | 提交截止（T+70 23:59）| 材料校验；系统锁定 | 截止提醒 |
| 评审期 | T+71 | T+84 | 初评（T+71–78）；终评（T+79–84）| 评审工作台开放 | 无（保密）|
| 公示期 | T+85 | T+91 | 获奖名单公示 | 申诉通道开放 | 公示公告 |
| 颁奖期 | T+92 | T+105 | 线上/线下颁奖；奖金发放 | 合同签署；转账 | 颁奖通知 |

> 日期填写实际日历日期时，以 `YYYY-MM-DD` 格式。

## 第 4 章：分阶段数据明细

### 4.1 创建活动

- **活动标题**：{正式标题}
- **背景故事**（3–5句）：企业痛点 / 团委政策背景，解释为什么现在发起这个活动
- **任务描述**：按赛道分段，各赛道任务书摘要（2–3段）
- **参赛约束**：个人参赛限制 / 组队规则 / 指导教师要求 / 企业身份限制（如 enterprise 档案禁止主办方员工参赛）
- **FAQ**（5–8条 Q&A，覆盖常见疑问：报名资格 / 保密规则 / 成果权属 / 奖金税务 / 提交格式）
- **IP/保密条款摘要**（Enterprise）或 **开源协议声明**（Youth-league）

### 4.2 发布与曝光

- **发布渠道与曝光数据**：

| 渠道 | 曝光 UV | 点击详情页 | 开始报名 | 完成报名 | 转化率 |
|------|--------|----------|---------|---------|--------|
| 官网首页推荐 | 12,300 | 1,840 | 620 | 317 | 2.6% |
| 微信公众号 | 8,200 | 960 | 280 | 142 | 1.7% |
| 掘金/CSDN/V2EX | 5,600 | 780 | ... | ... | ... |
| ... | | | | | |

- **整体漏斗**：总曝光 → 详情页访问 → 报名开始 → 报名完成
- **公告示例**：1条正式发布公告原文（含标题、正文、CTA链接占位）

### 4.3 报名与组队

- **总报名**：X人（赛道A: XX% / 赛道B: XX%）
- **地域分布**：Top 5 城市 + 百分比
- **经验分层**：学生 / 初级（0–2年）/ 中级（3–5年）/ 高级（5年+）/ 不填写
- **组队形成**：平台自动匹配 vs 自组队比例；中位组队时间（天）
- **队伍样例**（8–12支）：

| 队伍名 | 赛道 | 成员数 | 组建方式 | 队长背景 |
|--------|------|--------|---------|---------|
| 量子跃迁 | 技术 | 4 | 平台匹配 | 算法工程师，3年经验 |
| ... | | | | |

### 4.4 互动与运营

- **汇总数据**：帖子总数 X / 评论总数 X / 点赞总数 X
- **官方活动**：AMA 场次、直播、答疑时间表
- **帖子样例**（≥12条，格式统一）：

```
[帖子类型] 作者 | 时间
标题或内容摘要（100字以内）
点赞 X | 评论 X | 收藏 X
```

类型须覆盖：官方公告、技术问答、经验分享、情绪/抱怨、组队招募、导师答疑、庆祝祝贺、申诉质疑

### 4.5 作品提交

- **提交统计**：总报名队伍数 / 实际提交 / 提交率 / 最后24小时提交占比
- **物料分布**：repo占比 / 视频占比 / demo占比 / 文档占比
- **作品样例**（≥8个）：

| 队名 | 赛道 | 物料类型 | 描述摘要 | 占位链接 |
|------|------|---------|---------|---------|
| 量子跃迁 | 技术 | repo+视频+文档 | 基于LLM的质检异常检测系统 | [repo](https://github.com/synnovator-demo/quantum-leap) / [视频](https://demo.synnovator.io/v/ql-2025) |

### 4.6 评审与打分

- **评审名单**（4–6人）：

| 姓名（虚构）| 角色 | 所属机构 | 专业背景 |
|-----------|------|---------|---------|
| 王志远 | 主评审 | 智海科技 CTO 办公室 | 系统架构，15年 |

- **评分维度与权重**：表格（维度 / 权重 / 描述 / 分值区间）
- **打分样例**（至少5个作品，每个作品一行）：

| 队名 | 技术可行性(30%) | 落地性(25%) | 创新性(25%) | 完整度(20%) | 加权总分 |
|------|--------------|-----------|-----------|-----------|--------|
| 量子跃迁 | 87 | 82 | 79 | 91 | **84.4** |

- **评语样例**（3条，含正面和建设性反馈各一）

### 4.7 公示与申诉

- **公示期开始日期**，**公示内容**（获奖名单片段，表格形式）
- **申诉案例**（≥2个）：

```
案例 4.7.X：{案例名}
触发：{申诉方 + 申诉理由}
处理：{平台/主办方处理步骤，附时间线}
结果：{最终决定 + 依据 + 通知方式}
```

### 4.8 颁奖与发放

- **获奖名单**（按赛道排名，含队名 / 作品名 / 奖励内容 / 发放状态）
- **发放流程**：签约 → 开票 → 转账/颁奖，各步骤时间节点
- **税务/合同说明**（占位说明，如"个人所得税按劳务报酬预扣20%；框架合同待签"）
- **后续合作线索**（Enterprise）：如企业导师跟进 / 试点立项 / 实习录用意向；（Youth-league）：如参赛经历认定/指导教师荣誉证书

## 第 5 章：异常与边界案例库

每类案例格式：

```markdown
### 案例 5.X：{案例名}

**类型**：{涉密 / 抄袭 / 刷票 / 利益冲突 / 侵权 / 内容违规 / 系统故障 / 主办方违约 / 资格造假}
**涉及角色**：{参赛队伍 / 评审 / 主办方 / 平台}
**触发条件**：{具体场景描述}
**处理流程**：{平台/主办方响应步骤，含时间线}
**最终结果**：{处置决定 + 依据 + 公告方式}
**平台策略说明**：{为什么这样处理 / 参考的行业规则}
```

| 提交率 | 实际提交/已组队 | 60% | 48% | 80% | ... | ... |
